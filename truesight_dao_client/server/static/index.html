DAO Events Reference

Use this reference to determine which event type to submit. Each event type has specific required fields and when-to-use rules.

SALES EVENT

When to use: A bag is sold to an end customer (retail sale). Money changes hands. QR status will be updated to SOLD.
Required: Item (QR code), Sales price  |  Optional: Owner email, Stripe Session ID, Shipping Provider, Tracking number, Sold by, Cash proceeds collected by
[SALES EVENT] - Item: 2024OSCAR_20260121_32 - Sales price: 17.50 - Sold by: Gergana - The Way Home Shop - Cash proceeds collected by: Gary Teh

INVENTORY MOVEMENT

When to use: A bag moves between known holders in the supply chain (e.g., from warehouse to retailer, or between facilities). No end-customer sale.
Required: Manager Name (sender), Recipient Name, Inventory Item (SKU), QR Code, Quantity
[INVENTORY MOVEMENT] - Manager Name: Kirsten - Recipient Name: Gergana - The Way Home Shop - Inventory Item: Ceremonial Cacao Kraft Pouch - Oscar 2024 - QR Code: 2024OSCAR_20260121_32 - Quantity: 10

CONTRIBUTION EVENT

When to use: A DAO member contributes time, money, or resources. Used for TDG scoring and contribution tracking.
Required: Title, Body, Contributors  |  Optional: Amount, Type (Time/USD), TDG Issued, PR URLs
[CONTRIBUTION EVENT] - Title: Updated landing page docs - Body: Added DAO Events Reference section - Contributors: Sophia Truesight - Amount: 45 - Type: Time (Minutes)

PURCHASE AGREEMENT

When to use: A purchase agreement is signed with a farmer or supplier for future cacao delivery.
Required: Farmer Name, Agreement Date, Quantity, Price  |  Optional: Contract ID, Notes
[PURCHASE AGREEMENT] - Farmer Name: Maria Santos - Agreement Date: 2026-06-01 - Quantity: 500 - Price: 12.00/kg

QR CODE BATCH

When to use: A new batch of QR codes is generated and assigned to a specific SKU and manager.
Required: Batch ID, SKU, Quantity, Manager  |  Optional: Notes
[QR CODE BATCH] - Batch ID: 2026Q2-OSCAR - SKU: Ceremonial Cacao Kraft Pouch - Oscar 2024 - Quantity: 50 - Manager: Kirsten

DAO Query Endpoints

Read-only query endpoints for looking up transactions, QR codes, and inventory movements. All name fields support case-insensitive substring matching.

GET /dao/transactions — Sales transactions ▶
Parameters: partner (substring), sku (substring), from_date, to_date, limit (default 100, max 1000)
curl "https://edgar.truesight.me/dao/transactions?partner=soha" { "results": [ { "date": "2026-01-15", "partner": "SOHA", "sku": "Ceremonial Cacao Kraft Pouch - Oscar 2024", "qty": 10, "value": 340.00, "source_sheet": "QR Code Sales" } ], "count": 1 }
GET /dao/qr-codes — QR code lookup ▶
Parameters: manager (substring), owner (substring), sku (substring), status (exact), limit
curl "https://edgar.truesight.me/dao/qr-codes?manager=the+way+home" { "results": [ { "qr_code": "2024OSCAR_20260121_32", "sku": "Ceremonial Cacao Kraft Pouch - Oscar 2024", "status": "ON CONSIGNMENT", "manager": "Gergana - The Way Home Shop", "ledger": "AGL4" } ], "count": 10 }
GET /dao/inventory-movements — Track through hands ▶
Parameters: person (substring), role (sender/recipient), from_date, to_date, limit
curl "https://edgar.truesight.me/dao/inventory-movements?person=soha" { "results": [ { "date": "2026-01-15", "qr_code": "2024OSCAR_20260121_32", "sender": "Kirsten", "recipient": "SOHA", "note": "Restock" } ], "count": 21 }
